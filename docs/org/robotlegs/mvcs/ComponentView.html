Classpublic class ComponentView
InheritanceComponentView flash.display.MovieClip
Implements IComponentView

Abstract IComponentView implementation

Subclass this view class and a ComponentViewMediator will automatically be created.

  • Override the methods to add animation to the start and end phase of the view.
  • Call the convenience methods to add, swap or remove views.
  • Get notified when other views are created.
  • Release your listeners for garbage collection in the preDestroy or destroy method.

mapButtonToView()method 
protected function mapButtonToView(button:Object, viewClass:Class, listener:Function, swapView:Boolean = true, ... args):void

Map a button to a create a new view class. The onButtonClicked method will check if the clicked button exists in the buttons dictionary, set the class and parameters properties and dispatch a swap- or addview signal.

The listener will be realeased for garbage collection when the view is removed from the stage

Parameters

viewClass:Class — The class for the new instance
 
listener:Function — The listener to be called at mouse click
 
swapView:Boolean (default = true) — If the view should replace another view or be added
 
... args — Optional parameters passed to the constructor of the new view

onButtonClicked()method 
protected function onButtonClicked(event:MouseEvent):void

As default this method dispatches a button clicked signal to ComponentViewMediator, which calls its abstract button clicked listener.

If a button is mapped to a view class the _viewClassRequest and _viewParameters properties are set The mediator will ask the injected view for these values before dispatching a create view signal.

Parameters

event:MouseEvent — Native flash MouseEvent
